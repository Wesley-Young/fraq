`@fraqjs/plugin-ai` 提供了配置模型提供商和 API Key 的统一入口，方便基于 [Vercel AI SDK](https://ai-sdk.dev/) 的 AI 插件开发者访问用户开发者配置的语言模型。

## 安装与配置

将插件添加至 `dependencies`，然后在创建 `Context` 时引入并配置插件：

```typescript
import AiPlugin from '@fraqjs/plugin-ai';

ctx.install(AiPlugin, {
  // 在这里声明多个 AI SDK 提供商，每个提供商下配置多个模型
  providers: {
    // 提供商的名称可以自定义，建议使用 SDK 包名中的关键词以示区分
    deepseek: {
      // 目前支持自动配置的 SDK 包括：
      // - `@ai-sdk/anthropic`
      // - `@ai-sdk/deepseek`
      // - `@ai-sdk/google`
      // - `@ai-sdk/openai`
      // 在这里声明 SDK 之后，你需要手动将对应的 SDK 添加到项目的 `dependencies` 中
      sdk: '@ai-sdk/deepseek',
      // 这里用于提供传入 SDK 的配置项
      // 基本的配置项有：
      // - `apiKey`：必需，用户的 API Key
      // - `baseURL`：可选，API 的基础 URL，默认为 SDK 内置的 URL
      // 例如，DeepSeek 的默认 URL 是 `https://api.deepseek.com/v1`
      // 不同 SDK 能接受的配置项不同，具体请参考对应 SDK 的文档
      options: {
        apiKey: 'ds-xxx',
      },
      // models 是一个字符串数组，用于声明用户想要使用的模型名称
      // 具体模型名称需要参考对应提供商 / SDK 的文档
      // 例如 DeepSeek 的模型名称包括 `deepseek-v4-flash`、`deepseek-v4-pro` 等
      models: ['deepseek-v4-flash', 'deepseek-v4-pro'],
    },
    'openai-custom': {
      sdk: '@ai-sdk/openai',
      options: {
        apiKey: 'sk-xxx',
        // 可以通过 `baseURL` 配置自定义的 API URL
        // OpenAI 风格的 API 通常以 `/v1` 结尾
        baseURL: 'https://my-openai-api.com/v1',
      },
      models: ['gpt-5.5', 'gpt-5.5-mini'],
    },
    // 也可以直接在 providers 中提供模型实例
    moonshot: {
      'kimi-k2.5': (await import('@ai-sdk/moonshotai')).moonshot('kimi-k2.5'),
    },
  },

  // 在这里为模型配置别名，方便在插件中使用更友好的名称访问模型
  // 键 (key) 为别名，值 (value) 为 providers 中声明的模型名称，格式为 `提供商名称/模型名称`
  // 例如 `fast` 对应 `deepseek` 提供商下的 `deepseek-v4-flash` 模型
  aliases: {
    fast: 'deepseek/deepseek-v4-flash',
    'fast-gpt': 'openai-custom/gpt-5.5-mini',
    balanced: 'deepseek/deepseek-v4-pro',
    expert: 'openai-custom/gpt-5.5',
  },

  // 默认模型，插件中通过 `ctx.ai.model()` 访问时，如果没有指定模型名称，就会使用这个默认模型
  // 可以使用别名或者 `提供商名称/模型名称` 的格式指定默认模型
  // 如果没有配置这一项，默认使用 providers 中第一个提供商的第一个模型作为默认模型
  defaultModel: 'fast',
});
```

{/* prettier-ignore-start */}

<Callout>
`ai` 在不同的 provider 之间共享全局状态，请确保你的项目中只有一份 `ai` 实例。本插件将 `ai` 声明为 `peerDependencies`，你需要在自己的项目中将它添加到 `dependencies`。

此外，如果你通过直接声明 SDK 名称的方式配置了模型提供商，也需要将对应的 SDK 包添加到 `dependencies` 中。

</Callout>
{/* prettier-ignore-end */}

如果你是插件开发者，请将本插件添加到项目的 `peerDependencies` 中，并在自己的插件中声明依赖：

```typescript
import { definePlugin } from '@fraqjs/fraq';
import { AiService } from '@fraqjs/plugin-ai';

definePlugin({
  name: 'my-plugin',
  inject: {
    ai: AiService,
  },
  apply(ctx) {
    // 通过 ctx.ai.model 访问配置好的模型
  },
});
```

此外，你还需要将 `ai` 添加到 `peerDependencies` 中，以便使用其中的函数。

## 获取模型实例

插件会将用户配置的模型实例注入到 `ctx.ai` 中，你可以通过 `ctx.ai.model` 方法来获取模型实例：

```typescript
const model = ctx.ai.model(); // 获取默认模型，即用户配置中的 defaultModel，或者 providers 中第一个模型
const fastModel = ctx.ai.model('fast'); // 通过别名获取模型
const kimiModel = ctx.ai.model('moonshot/kimi-k2.5'); // 通过提供商名称/模型名称的格式获取模型

ctx.ai.hasModel('fast'); // true
ctx.ai.hasModel('nonexistent'); // false

ctx.ai.models(); // 获取所有模型的名称列表
```

## 调用 AI 功能

Vercel 的 `ai` 包提供了丰富的函数来调用语言模型，只需要传入模型实例和相应的参数即可。下面是一些简单的例子，更多用法请参考 [Vercel AI SDK 的文档](https://ai-sdk.dev/docs)。

### 生成文本

```typescript
import { generateText } from 'ai';

const { text } = await generateText({
  model: ctx.ai.model(),
  prompt: '用一句话介绍一下 Fraq。',
});

console.log(text);
```

### 流式生成文本

`streamText` 会返回一个流式结果，你可以通过 `textStream` 异步迭代器逐段读取生成的文本：

```typescript
import { streamText } from 'ai';

const result = streamText({
  model: ctx.ai.model(),
  prompt: '写一首关于海的短诗。',
});

for await (const delta of result.textStream) {
  process.stdout.write(delta);
}
```

### 生成结构化数据

`generateText` 和 `streamText` 都支持通过 `output` 选项来指定生成结果的格式：

```typescript
import z from 'zod';

const { output } = await generateText({
  model: ctx.ai.model(),
  output: Output.object({
    schema: z.object({
      recipe: z.object({
        name: z.string(),
        ingredients: z.array(
          z.object({ name: z.string(), amount: z.string() }),
        ),
        steps: z.array(z.string()),
      }),
    }),
  }),
  prompt: '请生成一份阳春面的食谱。',
});
```

这里输出的 `output` 将会是一个符合指定 Zod schema 的对象。

<Callout>
同样的，如果你在插件中使用 `zod`，也需要将 `zod` 添加到 `peerDependencies` 中。
</Callout>
{/* prettier-ignore-end */}

### 工具调用

`generateText` 和 `streamText` 都支持工具调用（Tool Calling），你可以通过 `tools` 选项让模型在生成过程中调用你定义的函数：

```typescript
import { generateText, tool } from 'ai';
import z from 'zod';

const { toolResults } = await generateText({
  model: ctx.ai.model(),
  prompt: '东京现在的天气怎么样？',
  tools: {
    weather: tool({
      description: '查询某个城市的天气',
      inputSchema: z.object({
        city: z.string().describe('城市名称'),
      }),
      execute: async ({ city }) => `${city}今天是晴天`,
    }),
  },
});

console.log(toolResults);
```
